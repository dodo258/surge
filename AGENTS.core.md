# AGENTS.core.md — Agent 协作、接管协议、运行期守卫

> 这是多 Agent 行为规范层。
> 适用于：主 agent / 子 agent / 模型切换 / 新会话接管。

---

## 一、TAKEOVER 硬闸（强制执行，无例外）

**触发条件（满足任意一项即触发）：**
- 新会话启动
- 模型切换（如 claude → gpt → claude）
- 上下文置信度 LOW
- 用户明确要求"接管"

**触发后必须按序执行：**

```
STEP 1: read(PROJECT.md)         → 获取项目身份、架构、约束
STEP 2: read(NOW.md)             → 获取当前任务、上一步、阻塞
STEP 3: read(memory/今日.md)      → 获取近期事件
STEP 4: 输出 TAKEOVER_SUMMARY    → 见下方格式
STEP 5: 才允许执行用户任务
```

⚠️ **STEP 4 之前，禁止回答任何实质性任务。**

**TAKEOVER_SUMMARY 输出格式：**

```
[TAKEOVER]
项目：{PROJECT.md → 项目名 | 无}
状态：{PROJECT.md → 状态}
当前任务：{NOW.md → 正在做什么}
上一步：{NOW.md → 上一步完成了什么}
下一步：{NOW.md → 下一步立即要做什么}
阻塞：{NOW.md → 当前阻塞 | 无}
置信度：HIGH(文件齐全) / MED(文件部分缺失) / LOW(无文件/信息不足)
```

置信度 **LOW** → 必须向用户说明："我没有足够的项目背景，请确认或提供"。

## 二、运行期守卫（每次关键响应后自检）

**每次完成一个关键响应后，内部执行以下自检：**

```
GUARD_CHECK:
  [ ] 本次回答是否体现了项目感知？
  [ ] 是否偏离了 NOW.md 描述的当前任务？
  [ ] 是否出现明显上下文遗失迹象（答非所问、重复建议已否决方案）？
```

**如果任意一项为否：**
1. 主动触发 memory_search
2. 重新加载 PROJECT.md + NOW.md
3. 重新生成响应，并说明"我重新确认了背景"

---

## 三、子 Agent 协作规范

- 子 agent 产出的关键结果必须由主 agent 写入 memory
- 子 agent 不得自行修改 PROJECT.md / NOW.md
- 主 agent 负责维护"单一真实来源"

---

## 四、模型切换注意事项

模型切换时（如 claude → gpt），接手模型必须：
1. 不假设前任模型做了什么
2. 必须重新执行 TAKEOVER 硬闸
3. 如果 NOW.md 未更新，必须询问用户确认当前状态

---

## 五、通用交付检查（强制 - 所有领域）

**任何交付物（代码/配置/文档/分析）执行前必须自检：**

```
DELIVERY_CHECK:
  [ ] 完整性 - 是否遗漏关联文件/模块？
  [ ] 一致性 - 多格式/多平台是否同步？
  [ ] 正确性 - 语法/格式/逻辑是否正确？
  [ ] 可用性 - 用户拿到后能否直接使用？
  [ ] 文档性 - README/注释是否同步更新？
  [ ] 预判性 - 用户可能的下一步需求？
  [ ] 冗余性 - 是否有重复/冲突/无用内容？
```

**执行方式：**
- 口头汇报检查清单结果
- 高优先级项目可要求用户确认每项

**违反则：**
- 必须补做检查后重新交付
- 重复违反则更新 SELF_IMPROVEMENT.md 强化记忆

## 六、主动预判机制（强制）

**每个任务完成后，必须主动提出：**

```
PROACTIVE_CHECK:
  1. "这个改动需要同步到其他文件吗？"
  2. "用户可能还需要什么相关功能？"
  3. "有什么潜在问题我没考虑到？"
  4. "这个方案是最佳实践吗？有更好的吗？"
```

**产出：**
- 至少提出 1 个预判或改进建议
- 即使用户没要求

---

## 七、Git 强制同步协议（硬性 - 每次会话必须执行）

**每次会话结束前，必须完成以下并汇报：**

```
GIT_SESSION_REPORT:
  [ ] git status -s  # 检查本地状态
  [ ] 列出修改的文件数量
  [ ] 列出未跟踪的新文件
  [ ] 如果有待提交 → git add -A && git commit
  [ ] 如果有待推送 → git push
  [ ] 汇报: "本次修改: X 文件, 已提交: 是/否, 已推送: 是/否"
  [ ] 询问用户: "需要我提交并推送吗？"
```

**汇报格式（必须输出）：**

> **本次会话 Git 状态：**
> - 修改文件: [列表或数量]
> - 新文件: [列表或数量]
> - 已提交: 是/否
> - 已推送: 是/否
> - 状态: 干净/有待处理
>
> **[需要我提交并推送吗？]**

**违反则：**
- 视为交付失败
- 立即停止当前任务
- 补做 git 操作后再继续

## 八、模式学习与传承

**每次交互自动记录到：**
- `SELF_IMPROVEMENT.md` - 错误与教训
- `USER_PATTERNS.md` - 用户偏好
- `DOMAIN_KNOWLEDGE.md` - 领域知识

**换模型时：**
- 新模型自动读取上述文件
- 完整继承经验与偏好
